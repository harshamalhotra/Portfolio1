BAKSA
A responsive web application to collect information and store it at one location.
DESCRIPTION
Baksa is a SaaS (Software as a Service) web application which allows people to collect information and store it in a single place. A person can create simple documents, store images, links to websites, and collaborate with others.
SCOPE OF THE PROJECT
To design a website:
  • To create simple documents.
  • To store images.
  • To save and store link to websites.
  • To be able to collaborate with others on the information stored.
We want to do by making sure the experience is simple and the processes are intuitive and streamlined.
RESEARCH

In the first stage, I conducted a competitive analysis among Evernote, Google Keep and Things. Next, I wanted to understand my potential users. The key goal was here to become familiar with the potential users and determine their requirements. I created a survey and asked them questions about how they were currently saving their content and writing content.I gained a lot of useful information through all the responses of participants. Some of the most important insights that I gained for my application are as follows:

  • The website needs to be responsive because all the potential users were using their laptop and their smartphones to manage their content.
  • The website needs to autosave and should be able to save the changes made to their content synchronously.
  • The website needs to simple to use so that the users can focus on their work instead of learning to figure out the product.
  • My website should have a social sign on because all of my respondents agreed to sign in through their other accounts as it saves time.
  • I also found out that reminders can be a possible features that I can incorporate in this product in the future.

I gathered information from my potential users and prepared user personas.

PERSONA

IDENTITY AND BRANDING

I wanted my product to be identified as a box to store all the content. It should be able to store images and text. I played all around with various variations of the word “box” and decided to call it “baksa”. A box is called “baksa” in hindi.

I wanted baksa to have a vibe which is professional, trustworthy and slightly playful to engage the users. I decided to use Helvetica Neue and Garamond as font families, both of which are classic fonts and are in complete harmony with each other.

USER STORIES AND USER FLOWS

Based on business requirements and our research, I determined a list of user stories that will help in creation of user flows.
WIREFRAMES

After designing user flows, I then designed started designing low fidelity wireframes using Balsamiq. I created four versions of the home page for baksa in which I played around with different spatial placements of the various elements.

I then conducted a preference test with 10 people. Seven out of ten people preferred version 1 of the hero page. I kept the first one for the low fidelity wireframe and then went ahead and created the other low fidelity wireframes for baksa.

By now, I had an idea on how baksa will be roughly structured. I then created medium fidelity wireframes by adding content to it. I designed these without color because I did not want my test users to get distracted with the color template that I use.

I then uploaded my medium fidelity prototype onto peek to understand how users perceived it. I received very positive feedback. The user appreciated the clean design and focused on how clean the hero page looked.

The usability test results motivated me to go ahead and I created the high fidelity wireframes for baksa by adding color and content to it.

DEPLOYEMENT

I deployed baksa on Gitbhub after coding it using HTML and CSS. Baksa was supposed to be a responsive website. The use of media queries to facilitate the responsiveness was another challenge that I had to overcome. While testing my code in Chrome on a retina display, I had a lot of problems. I then ultimately figured out that Chrome has testing tools which can test the way baksa will look in different displays.

Baksa on github
USABILITY TEST

I asked three persons how they felt about baksa’s design. They really appreciated the use of bold colors and thought the site looked easy to use. They especially liked the display of the items on the dashboard.

The users thought that they would like to have more functionality in the product. They also wanted the sign out option to be more visible.

LESSONS LEARNT

Baksa was the first project that I had completed at Bloc. It was a great learning curve for me. I used softwares like Balsamiq, Sketch, Photoshop and Illustrator for the very first time. I also used HTML and CSS to deploy baksa on github.

While creating medium fidelity wireframes, using Illustrator was a great challenge for me. To include responsiveness by using media queries in css, was another trial for me.

I feel I could have reached the final stage of coding much faster, if I would have skipped the step of creating medium fidelity wireframes. All in all, creating baksa infused a lot of confidence in me and taught me to use a lot of design tools.
